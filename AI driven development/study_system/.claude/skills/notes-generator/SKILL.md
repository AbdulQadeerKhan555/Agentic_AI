# Notes Generator

## Overview

The notes-generator creates comprehensive study notes structured according to Bloom's Taxonomy, a hierarchical framework for organizing learning objectives across six cognitive levels. This skill transforms topic information into pedagogically sound notes that progress from basic recall to advanced synthesis and evaluation.

Notes generated by this skill are stored in the `notes/` directory and follow a systematic structure that facilitates deep learning and retention. The skill accommodates both scenarios: generating notes from a topic title alone, or incorporating content from provided reference files.

## Core Principles

### Bloom's Taxonomy Framework

The notes-generator structures content according to the revised Bloom's Taxonomy, which organizes cognitive processes into six hierarchical levels:

1. **Remember**: Recall facts, terms, basic concepts, and answers
2. **Understand**: Explain ideas or concepts in one's own words
3. **Apply**: Use information in new situations and contexts
4. **Analyze**: Draw connections, identify patterns, distinguish components
5. **Evaluate**: Justify decisions, critique arguments, assess value
6. **Create**: Produce original work, design solutions, synthesize information

Each level builds upon the previous, creating a scaffold for progressively deeper understanding.

### Pedagogical Structure

Notes generated follow this hierarchical organization:
- Lower-order thinking skills (Remember, Understand) establish foundational knowledge
- Middle-order skills (Apply, Analyze) develop practical competence
- Higher-order skills (Evaluate, Create) foster critical thinking and innovation

## Note Generation Process

### Step 1: Topic Analysis

When a user requests note generation, first analyze the topic:

**Extract Key Information:**
- Topic title and scope
- Subject domain (e.g., computer science, biology, history)
- Intended depth (introductory, intermediate, advanced)
- Reference materials provided (if any)

**Determine Structure:**
- Identify main concepts and subtopics
- Establish logical progression
- Plan section organization

### Step 2: Reference File Processing

If reference files are provided:

**Read and Analyze:**
- Use the `Read` tool to examine reference content
- Extract key concepts, definitions, and examples
- Identify relationships between ideas
- Note important facts, figures, and citations

**Synthesize Information:**
- Organize extracted content by relevance
- Identify gaps requiring additional explanation
- Determine which taxonomy levels the source material addresses
- Plan how to structure content across all six levels

### Step 3: Content Development by Taxonomy Level

Generate content for each Bloom's Taxonomy level:

#### Level 1: Remember

**Purpose**: Establish foundational knowledge through recall

**Content to Include:**
- Key terms and definitions
- Important facts and dates
- Basic concepts and principles
- Fundamental formulas or rules
- Essential vocabulary

**Format:**
```markdown
## Remember: Foundational Knowledge

### Key Terms
- **Term 1**: Definition
- **Term 2**: Definition

### Essential Facts
- Fact 1
- Fact 2

### Core Concepts
Brief explanations of fundamental ideas
```

#### Level 2: Understand

**Purpose**: Demonstrate comprehension through explanation

**Content to Include:**
- Explanations in accessible language
- Summaries of complex ideas
- Comparisons and contrasts
- Interpretations of data or concepts
- Paraphrased information

**Format:**
```markdown
## Understand: Conceptual Comprehension

### Concept Explanations
Detailed explanations of how things work or why they matter

### Comparisons
- Similarity/difference analysis
- Analogies and metaphors

### Interpretations
What the information means in context
```

#### Level 3: Apply

**Purpose**: Demonstrate practical use of knowledge

**Content to Include:**
- Worked examples and solutions
- Step-by-step procedures
- Application scenarios
- Practice problems
- Implementation guidelines

**Format:**
```markdown
## Apply: Practical Application

### Worked Examples
Step-by-step demonstrations

### Application Scenarios
Real-world contexts where concepts apply

### Practice Exercises
Problems for applying learned concepts
```

#### Level 4: Analyze

**Purpose**: Break down information and identify relationships

**Content to Include:**
- Component analysis
- Pattern identification
- Cause-and-effect relationships
- Structural breakdowns
- Comparative analysis

**Format:**
```markdown
## Analyze: Critical Examination

### Component Analysis
Breaking down complex systems into parts

### Patterns and Relationships
Connections between ideas

### Comparative Analysis
Detailed comparisons revealing insights
```

#### Level 5: Evaluate

**Purpose**: Make judgments based on criteria

**Content to Include:**
- Critical assessments
- Advantages and disadvantages
- Justifications and rationales
- Critiques of approaches
- Evidence-based conclusions

**Format:**
```markdown
## Evaluate: Critical Assessment

### Strengths and Weaknesses
Balanced evaluation of approaches

### Justifications
Reasoning behind conclusions

### Critical Perspectives
Different viewpoints and their validity
```

#### Level 6: Create

**Purpose**: Synthesize knowledge into original work

**Content to Include:**
- Synthesis of multiple concepts
- Novel applications or solutions
- Design proposals
- Original frameworks
- Integration of ideas

**Format:**
```markdown
## Create: Synthesis and Innovation

### Integrated Understanding
How concepts combine into larger frameworks

### Novel Applications
New ways to apply knowledge

### Original Insights
Unique perspectives or solutions developed
```

### Step 4: Note Structure Assembly

Assemble the complete note document:

**Header Section:**
```markdown
# [Topic Title]

**Subject**: [Domain]
**Date Created**: [YYYY-MM-DD]
**Taxonomy Level**: All Six Levels
**Reference Materials**: [List if applicable]

---
```

**Body Sections:**
- Include all six taxonomy levels in order
- Ensure smooth transitions between levels
- Maintain consistent formatting
- Add visual separators between major sections

**Footer Section:**
```markdown
---

## Additional Resources

- Reference to related notes
- External resources for further study
- Practice materials or exercises

## Review Schedule

- Initial review: [Date]
- First reinforcement: [Date + 1 day]
- Second reinforcement: [Date + 3 days]
- Third reinforcement: [Date + 7 days]
```

Generate the note file:

**File Naming Convention:**

- Use kebab-case for topic names

## Content Guidelines

### Writing Style

**Clarity and Precision:**
- Use clear, unambiguous language
- Define technical terms on first use
- Provide concrete examples
- Avoid unnecessary jargon

**Academic Tone:**
- Maintain formal register as specified in CLAUDE.md
- Use objective, evidence-based language
- Employ appropriate epistemic modality
- Structure explanations logically

**Pedagogical Effectiveness:**
- Progress from simple to complex
- Build on previously introduced concepts
- Provide sufficient context
- Include practical applications

### Content Depth

**Foundational Levels (Remember, Understand):**
- Comprehensive coverage of basics
- Clear definitions and explanations
- Multiple examples where helpful
- Target: 30-40% of total content

**Application Level (Apply):**
- Practical, actionable examples
- Step-by-step procedures
- Real-world scenarios
- Target: 20-25% of total content

**Higher-Order Levels (Analyze, Evaluate, Create):**
- Deep analytical content
- Critical perspectives
- Synthesis and integration
- Target: 35-45% of total content

## Special Considerations

### Reference File Integration

When reference files are provided:

**Citation Practice:**
- Acknowledge source material appropriately
- Use inline references where applicable
- Maintain academic integrity
- Distinguish between source content and generated analysis

**Content Extraction:**
- Identify key information systematically
- Organize by taxonomy level
- Fill gaps with generated content

### Topic-Only Generation

When no reference files are provided:

**Knowledge Synthesis:**
- Draw upon general knowledge of the subject
- Provide foundational information
- Include standard examples and applications
- Note limitations of coverage

**Clarification Protocol:**
- If topic is ambiguous, use `AskUserQuestion` tool

- Verify scope and depth expectations
- Confirm subject domain
